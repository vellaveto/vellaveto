What It Does

Four layers of the Vellaveto agentic security control plane.

Govern

Centralize policy, approvals, tenant controls, and access decisions instead of scattering guardrails across SDKs and apps.

Enforce

Run inline on stdio, HTTP, WebSocket, and gRPC paths so tool calls are evaluated before they reach a live system.

Discover

Inventory MCP servers, track topology drift, and fail closed on unknown or ambiguous tools before they slip into production.

Prove

Produce tamper-evident audit, SIEM-ready telemetry, and evidence artifacts for incident response, compliance, and forensics.

Try it

Evaluate a tool call with a single request.

# Evaluate an MCP tool call against your policy
curl -X POST http://localhost:3000/api/evaluate \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "filesystem.write",
    "arguments": {
      "path": "/etc/passwd",
      "content": "..."
    },
    "context": { "agent": "coding-assistant" }
  }'

# Response: { "decision": "deny", "reason": "write to sensitive system path", ... }

Who it's for

Teams running AI agents that interact with real systems through MCP tool access.

  • Coding Assistants Guard file system, terminal, and git operations from unintended side effects.
  • Autonomous Agents Enforce boundaries on agents that chain tool calls without human oversight.
  • Enterprise AI Meet compliance requirements with auditable, policy-driven control over agent behavior.
